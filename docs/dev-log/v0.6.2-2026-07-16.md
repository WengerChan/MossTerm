# v0.6.2 · Pane 树持久化

> v0.5.8 抽出 paneTree.ts 算法层，v0.6.2 给 `useTabsStore` 加 zustand persist
> middleware，layout（tab / split / pane 树）跨 reload 不丢。

## 关键设计决策

### 1. 存什么 / 不存什么

- **存**：`tabs` 数组（id / title / panes 树 / activePaneId）+ `activeTabId`
- **不存**：
  - `Tab.sessionId` / `Tab.state` / `Tab.profileId` —— 运行时字段，跨进程早死
  - `Pane.sessionId`（递归 split 节点 children）—— 同上
- **partialize + onRehydrate 双清空**：保险（防 v0.6.0 之前脏数据 / 手动改 localStorage）

### 2. 启动行为

- reload 后所有 leaf `sessionId=null` → UI 走"未连接"占位（不调后端 Open）
- 用户重新 open 后再填 `pane.sessionId`
- **风险**：split 后某 leaf 已 null 但用户没注意 → 关掉它重建（v0.7.x 引入"恢复 layout 时提示哪些 pane 缺 session"）

### 3. 持久化 key

- `mossterm:tabs:v1` —— version 后缀方便未来 schema 变更
- zustand `persist` 配 JSON storage（走 localStorage；显式写出 setItem / getItem / removeItem 而非依赖默认实现，便于未来切到 IndexedDB）

## 关键 API

```typescript
// store
useTabsStore  // 包装了 zustand persist(middleware)
  - name: "mossterm:tabs:v1"
  - partialize: (state) => ({ tabs: stripRuntimeFields(state.tabs), activeTabId })
  - onRehydrateStorage: 再清一遍运行时字段
  - version: 1

// helpers（file-local）
function stripRuntimeFields(tabs: Tab[]): Tab[]     // 顶层 + 递归
function stripPaneSessionId(p: Pane): Pane          // 递归 split children
```

## 已知遗留

- 启动 layout 恢复时**不提示**哪些 pane 缺 session —— 用户得自己识别"占位"leaf（v0.7+）
- 持久化只 localStorage；多 tab 同步 / 设备间同步（v0.8+ 排期）
- 没加"reset layout" 入口（用户要清掉 layout 得手动 localStorage.removeItem；v0.7+ 加设置面板按钮）
