## 关键修复

### 1. `startCreate` id bug（致命）
- **v0.5.5 bug**: `id: \`tmp-${Date.now()}\`` → SaveProfile 走 UpdateProfile 路径 → 后端找不到 `tmp-XXX` → 报错
- **v0.5.6 fix**: `id: ""` → SaveProfile 走 AddProfile 路径

### 2. `ProfileEditModal` 容器 + 单一 modal 槽位
- 用 `useUIStore.modal.id === "profile-edit"` 匹配
- SessionForm 嵌入内部，关闭时调 `resetForm + closeModal`
- 单一 modal 槽位设计：避免 v0.5.5 老 `openModal({ id: \`delete-profile-${p.id}\` })` 的多槽位 bug

### 3. `ConfirmDeleteProfile` 模态
- 接收 `modal.props.profileId`（从 SessionTree 透传）
- 调 `useConnectionStore.deleteProfile`（v0.5.5 已接通）
- Toast 反馈成功/失败
- 删除中 disable 按钮 + dismissOnBackdrop 防止误关

### 4. `appStore.init` 启动拉 profile
- v0.5.5 appStore.init 啥都不拉（只有 TODO 注释）
- v0.5.6: 调 `useConnectionStore.refreshProfiles()` 启动时拉后端 profile 列表
- 动态 import 避免循环依赖（appStore 已被 connectionStore 间接依赖）

### 5. `SessionTree` 改用单 modal id
- v0.5.5: `openModal({ id: \`delete-profile-${p.id}\` })` 每次都建新 id
- v0.5.6: `openModal({ id: "confirm-delete-profile", props: { profileId } })` 复用同一槽位

## 完整流程

```
用户点 Sidebar "+" 按钮
  → useSessionStore.startCreate()
    → form.draft = emptyProfile (id="")
    → uiStore.openModal({ id: "profile-edit", title: "新建 profile" })
  → ProfileEditModal 检测 id 匹配 → 渲染 <SessionForm>
  → 用户填字段 → saveProfile(draft)
    → 后端 App.SaveProfile(draft) → id="" → AddProfile 路径
    → 后端返回成功
  → connectionStore.saveProfile 更新本地 state
  → SessionForm onClose → resetForm + closeModal
  → modal 关闭

用户点 SessionTree hover "编辑" 按钮
  → useSessionStore.startEdit(p)
    → form.draft = { ...p }
    → uiStore.openModal({ id: "profile-edit", title: `编辑 profile: ${p.name}` })
  → ProfileEditModal → SessionForm (editing)
  → 同上 save 流程（id 非空 → UpdateProfile 路径）

用户点 SessionTree hover "删除" 按钮
  → openModal({ id: "confirm-delete-profile", props: { profileId } })
  → ConfirmDeleteProfile 弹确认
  → 确认 → connectionStore.deleteProfile → 后端 App.DeleteProfile
  → Toast 反馈 → 关闭 modal
```

2. `npm run lint` 需要 `eslint` devDep + `.eslintrc`（v0.0 时代就没配）
3. `@types/xxx` import 路径在 strict 下报 TS6137（v0.5.0 B 之前就这样）
