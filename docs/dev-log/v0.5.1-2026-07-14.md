# v0.5.1 · SFTP 文件浏览器（前端 + 后端 wailsbindings）

> **两个并行子任务合并**：把 v0.5.0 留的 SFTP 后端接到 wailsbindings，前端写一个独立
> 浏览器 Modal。中间**挖出一个 v0.1 隐 bug**（OpenSession 顺序写反）。

## 两个子任务交付

### A · SFTP 暴露到 wailsbindings（后端）

**目标**：让前端能通过 `App.Sftp*` 调 SFTP，并把 SFTP client 生命周期绑到 session。

**实现要点**：
- `sshclient.Connector.RawClient()` —— 暴露底层 *ssh.Client
- `session.Session.Connector()` —— 让 app 从 session 拿到底层 connector
- `app.App`：
  - `sftpClients map[session.ID]*sftpclient.Client` 懒加载缓存
  - `sftpFor(sessID)` 私有方法（double-checked locking + lazy eviction）
  - `SftpClient(sessID)` 公开 wrapper（wailsbindings 跨包调）
  - `openSftp func(*ssh.Client) (*sftpclient.Client, error)` 测试钩子（默认 wrap `sftpclient.Open`）
- 7 个 SFTP binding：`SftpList` / `SftpStat` / `SftpMkdir` / `SftpRemove` /
  `SftpRename` / `SftpRead` / `SftpWrite`

**关键设计**：

1. **公开 `SftpClient` + 私有 `sftpFor`**：spec 把 `sftpFor` 标私有，但 wailsbindings
   在不同 package 调不到。加一行公开 wrapper 作 stable Go API，保持 `sftpFor` 内部性
2. **懒加载 + 复用**：pkg/sftp client 是 goroutine unsafe（文档明确），复用 map
   条目避免每次开 / 关 subsystem channel
3. **Lazy Eviction 生命周期**：session 关 → 下次 sftpFor 看到 state≠Established →
   删 map 条目 + 返回 error。不用 hook session 关闭事件、不用双簿记
4. **double-checked locking**：10 goroutine 并发拿同一 client，map 永远 ≤ 1 entry
5. **测试钩子 `openSftp`**：默认 `sftpclient.Open`，测试可注入 mock 跳过真 SFTP
   subsystem 协商

**意外收获 — v0.1 隐 bug 修复**：
- `sshclient.client.go` 的 `OpenSession` 把 `StdinPipe/StdoutPipe` 写在 `Shell()` 之后
- x/crypto v0.22.0 的 `Session.StdinPipe` 在 `s.started==true` 时返回
  `"StdinPipe after process started"`，而 `Shell()` 内部 `s.start()` 会把 `started`
  置 true
- 正确顺序是 StdinPipe/StdoutPipe → Shell
- **v0.1 起就有，但 sshclient 单测用 mock 不调 OpenSession；v0.5.1 写 in-process SSH
  server 第一次触发到**
- 已修；**建议 v0.5.1+ 写一个真 SSH server 端到端 integration test 守住这条路径**

### B · SFTP 浏览器（前端）

**目标**：独立 Modal（不集成 Pane 树，v0.6+ 再做），工具栏"SFTP 浏览器"按钮触发。

**实现要点**：
- `SftpBrowser.tsx`（937 行）—— 完整列目录 + 面包屑 + 文本查看 + 新建 / 删除 / 重命名
- `sftpBrowserStore.ts`（38 行）—— Zustand store 控制 open/close + 当前 sessionID
- `TabBar.tsx` 加"SFTP 浏览器"按钮（lucide-react `Folder` 图标）
- `App.tsx` 顶层挂载 `<SftpBrowser />`（受 store 控制）
- `App.d.ts` 扩展 7 个 Sftp* 方法声明（与后端 wailsbindings 一一对应）

**关键设计**：

1. **Modal 而非 Drawer**：避开既有 `SftpPanel` 右侧抽屉的位置冲突
2. **单实例浏览器**：`sftpBrowserStore.open(sid)` 直接覆盖 sessionID
3. **stale response 防护**：`reqTokenRef.current` 守卫 List 回调
4. **关浏览器 / 换 sessionID 各一个 useEffect**：职责清晰（开关 vs 切换）
5. **子对话框走本地 state**：不走 `uiStore.modal`（单 slot 会被多子对话框冲掉）
6. **5/7 binding 调用**：用 `List` / `Read` / `Mkdir` / `Rename` / `Remove`；`Stat` /
   `Write` 留接口（v0.5.1 范围外）

**踩坑 / 新发现**：
- 既有 `SftpPanel` 不动（MainLayout / Sidebar / CommandPalette 还在用），SftpBrowser
  走另一条 store 路径，**不**冲突
- 两个 `formatPerms` 重复：SftpBrowser 用 `& 0o7777`（兼容 setuid/setgid/sticky），
  `@utils/format.formatFileMode` 用 `& 0o777`。v0.6+ 统一
- path helpers（30 行）内联在 SftpBrowser，v0.6+ 拆到 `@utils/path`

**没有新依赖**：复用 `lucide-react` + `clsx` + `zustand` + `react`（v0.5.0 已装）。

## 7 个 SFTP binding 签名（wails 反射契约）

```go
// 1. 列目录（pageSize<=0 用默认 200；pageToken 暂未实现分页）
SftpList(ctx, sessionID, path string, pageSize int, pageToken string) (sftpclient.ListPage, error)
// 2. 取文件/目录元数据
SftpStat(ctx, sessionID, path string) (sftpclient.Entry, error)
// 3. 建目录（单层）
SftpMkdir(ctx, sessionID, path string) error
// 4. 删文件/空目录
SftpRemove(ctx, sessionID, path string) error
// 5. 改名/移动（同 mount）
SftpRename(ctx, sessionID, oldPath, newPath string) error
// 6. 读小文件（≤ 1 MiB 推荐）
SftpRead(ctx, sessionID, path string) ([]byte, error)
// 7. 写小文件（覆盖写）
SftpWrite(ctx, sessionID, path string, data []byte) error
```

**前端类型映射**：
- `[]byte` → `Uint8Array`
- `time.Time` → `string (RFC3339)`
- `os.FileMode (uint32)` → `number`
- `error` → `.catch()`

## 反思

1. **v0.1 隐 bug 暴露**：sshclient 单测用 mock 不调 OpenSession，导致 OpenSession 顺序
   错在 v0.1 ~ v0.5.0 期间一直存在。**in-process SSH server 的 integration test 是发现
   真 bug 的关键**——以后写 sshclient / sftpclient 改动都该跑这条路径
2. **公开 vs 私有方法**：wailsbindings 跨包访问是个常见痛点。`SftpClient` + 私有
   `sftpFor` 的双层设计保持内部实现可改 + 外部契约稳定
3. **测试钩子（`openSftp` 字段）**：把"真 SFTP 协商"和"app 集成逻辑"解耦，让单测
   不用真跑 SSH server 又能覆盖 sftpFor 全部路径。比直接用 interface mock 更轻
4. **Lazy Eviction 取代 session 关闭 hook**：不用双簿记、不用 Manager.OnClose 钩子，
   简单且正确
5. **前端 Modal vs Drawer 选型**：既有 `SftpPanel` 占右侧抽屉位置，SftpBrowser
   走中央 Modal；v0.6+ 整合时再决定要不要二选一
6. **v0.5.1 没做**：分页（pkg/sftp 限制）、大文件传输（断点续传 / 进度条）、Pane 树
   集成、upload UI、binary preview —— 留给 v0.5.2 / v0.6.0
