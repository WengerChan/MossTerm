# v0.5.8 · SFTP 集成到 Pane 树

> **双栏落地**：每个 SSH session 可挂 SFTP 子 pane（SplitPane 横/竖二分），
> 取代 v0.5.1 的独立 modal 弹出。Pane 树加 `kind: 'terminal' | 'sftp' | 'split'` 显式分流。

## 关键设计决策

1. **Pane 类型加 `kind` 字段**：旧实现靠 `split === null` 隐式判 leaf，v0.5.8 改为显式三态
2. **算法层抽离** `paneTree.ts`：pure functions（无 React/zustand 依赖），可单测
3. **SftpBrowserContent 抽离**：v0.5.1 ~700 行大组件拆成"业务 + modal wrapper + pane wrapper"三层
4. **addSftpPane 不自动 split**：直接平铺到 root，避免隐式 wrap（用户可手动 split）
5. **零新 npm 依赖**：测试用 Node 22+ `node:test` + `--experimental-strip-types`（项目已有 Node 26）
6. **零后端改动**：SFTP bindings v0.5.1 已齐全（`SftpList/Stat/Mkdir/...`），SFTP client 共享 SSH 连接

## 顺手修的 v0.5.0 B bug

- `closePaneFromTree` 在嵌套 split 关闭 + children lift 时会生成 3+ child 的 split 节点（违反"split 节点 = 2 child"软约束）
  → 修复：sub.length > 2 时平铺到 root 层
- `createSplit` 复用 `left.id`（注释留"调用方负责 id 重新分配"）导致唯一根误判
  → 修复：始终分配新 id（接受可选 `idFactory` 入参）
