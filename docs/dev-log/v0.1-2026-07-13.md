## 🔌 Wails 绑定方法（前端可调用的最终签名）

```go
// -----------------------------------------------------------------------------
// Session 相关
// -----------------------------------------------------------------------------
ListSessions(ctx context.Context)                                []session.Info
OpenSession(ctx context.Context, req session.OpenRequest)        (string, error)  // 返回 ID
CloseSession(ctx context.Context, id string, force bool)         error
SendInput(ctx context.Context, id string, data []byte)           error
ResizePTY(ctx context.Context, id string, cols, rows int)        error

// -----------------------------------------------------------------------------
// Profile CRUD
// -----------------------------------------------------------------------------
ListProfiles(ctx context.Context)                                []config.Profile
SaveProfile(ctx context.Context, p config.Profile)               error  // p.ID 空时报错；非空则 Add or Update
DeleteProfile(ctx context.Context, id string)                    error

// -----------------------------------------------------------------------------
// Secret 元数据（不含内容）
// -----------------------------------------------------------------------------
ListSecretsItems(ctx context.Context)                            []secret.Item
SaveSecret(ctx context.Context, name, kind, content string)      (string, error)  // 返回 ID
GetSecretContent(ctx context.Context, id string)                 (string, error)
```

**绑定规则**：
- 必须导出（首字母大写）
- `context.Context` 是首参（Wails 特殊处理）
- 参数 / 返回值必须是可导出类型或基本类型
- `[]byte` 在事件 payload 中会被转成前端 Uint8Array

## 🚀 启动流程（cmd/mossterm/main.go）

```
parseFlags() → *cliFlags
buildLogger() → *slog.Logger
  ↓
// 1. 加载 / 初始化配置
cfg, _ := config.New(flags.config)               // 不存在则写 Defaults() + 复制 config.example.toml
  ↓
// 2. 凭据存储：优先系统 keyring，失败自动降级到加密文件
sec, _ := secret.New({
  UseSystemKeyring: true,
  Argon2Params:     secret.DefaultArgon2Params,
})
  ↓
// 3. 跳板策略 registry
ag := agent.NewMemoryRegistry()                  // v0.1 空注册表
  ↓
// 4. 协议 connector registry
reg := connect.NewMemoryRegistry()
reg.Register("ssh", sshAdapter(sshclient.New))   // sshAdapter 适配签名
  ↓
// 5. 会话 manager（带 connectors 注入）
mm := session.NewMemoryManager().WithConnectors(reg)
  ↓
// 6. 装配 app
core := app.New(app.Deps{
  Cfg:        cfg,
  Secret:     sec,
  Sessions:   mm,
  Agents:     ag,
  Connectors: reg,
  Emitter:    wailsEmitter{},
  Log:        logger,
})
  ↓
// 7. 绑定层
api := wailsbindings.New(core)
  ↓
// 8. Wails 启动
go wails.Run(&options.App{
  OnStartup:        core.OnStartup,    // 存 ctx
  OnDomReady:       core.OnDomReady,    // emit "app:ready"
  OnShutdown:       core.OnShutdown,    // CloseAll sessions + cfg.Save + secret.Close
  Bind:             []any{api},
})
  ↓
select {
  case sig := <-sigCh:  // SIGINT / SIGTERM
    logger.Info("received signal, shutting down", "signal", sig.String())
    return nil
  case err := <-errCh:
    if err != nil {
      return fmt.Errorf("wails.Run: %w", err)
    }
    return nil
}
```

**启动期关键函数**：

| 函数 | 职责 |
|---|---|
| `sshAdapter(newFn)` | 把 `sshclient.New(connect.Deps) (*Connector, error)` 适配成 `connect.Factory` 签名（返回 interface 而非具体类型） |
| `isAlreadyRegistered(err)` | 字符串匹配吞掉 "already registered" 错误（重复启动测试时常见） |
| `wailsEmitter.Emit(ctx, event, data...)` | 包装 `wailsruntime.EventsEmit`，让 `app.App` 推送事件到前端 |

## 🧪 关键设计决策（v0.1 review 重点）

### 1. State 同步：`atomic.Int32` + `atomic.Pointer[Info]`
- 比 `sync.RWMutex` 更便宜（无锁读）
- 写：`s.state.Store(int32(st))`；读：`State() State { return State(s.state.Load()) }`
- 满足 "任意线程读 + 写" 的场景

### 2. Event fan-out：中央 `events` 通道 + `fanoutLoop`
- **不走直接 broadcast**：readLoop 不直接调 `broadcast` 推给 sub，避免 readLoop 因为某个慢 sub 而被阻塞
- **三段式**：readLoop → events (cap 64) → fanoutLoop → broadcast → 各 sub (cap 64)
- **`tryPublish` 是非阻塞 send**（`select + default`），events 满就丢
- **broadcast 内部**对每个 sub 做非阻塞 send，慢消费者就丢
- **实际启动了 4 个 goroutine**（readLoop / writeLoop / fanoutLoop / manager.Open 同步执行的 dial），与架构文档 "三个 goroutine" 的字面约定稍有偏差

### 3. 输入背压：`inputCh` cap 64 + `ErrInputFull`
- `Session.Input` 先检查 `state==Established`，否则拒绝
- `select` 投递 + `default` 分支返回 `ErrInputFull`（前端重发）
- v0.1 没做 4KB/16ms 输入批处理（v0.2 改进）

### 4. 连接失败 / 资源清理
- `Close` 用 `closeOnce` 保护，保证只执行一次
- 流程：setState(Closing) → tryPublish state 事件 → 持 connMu 写锁取出 sess/dialed 引用并置 nil → 释放锁 → close 引用 → close(done) 唤醒所有阻塞 goroutine → setState(Closed) → 关闭所有 sub channel
- **`sess` / `dialed` 用 `sync.RWMutex` 保护**（`connMu`），readLoop 持读锁拿到本地引用后**无锁**使用，避免持锁期间被 read/write syscall 阻塞
- Manager.Open 失败回滚：dial 失败 → 直接返回；opensession 失败 → `_ = conn.Close()`；start 失败 → `_ = s.Close(true)`（让 closeOnce 接管清理）

### 5. 避免循环依赖
- `connect/auth_convert.go` 是接收原始字段的 free function `AuthMethodFromSpec(kind, password, keyID, passphrase)`（不 import session）
- `session/auth_convert.go` 是 `AuthSpec` 上的薄包装方法 `(s AuthSpec).ToAuthMethod()`
- `DialParamsFrom` 放在 `session/dial_convert.go`（不能放 connect，会形成 `connect → session → connect`）

### 6. 协议装配点：`app/wire.go`
- 集中 import 所有具体实现包（sshclient），让 `app/app.go` 不直接依赖 sshclient
- `WireDefaultConnectors(reg)` 被 `app.New` 自动调用；`Deps.Connectors` 非空时跳过（允许 caller 预注入自己的 registry）
- "已注册" 错误用 `strings.Contains` 字符串匹配吞掉，避免破坏 "user 覆盖默认" 的语义
