# v0.6.0 · SFTP download streaming + SSH multi-hop + port forward

> **三主线合并**：
> 1. **下载流式（C）** — 复用 v0.5.10 上传架构，补齐 download 半边（分块 + 断点续传 + 校验）
> 2. **跳板链（A）** — `internal/agent/` 从脚手架 → 真实 `direct` / `single-jump` / `multi-hop` 三种 BuildFunc
> 3. **端口转发（B）** — `internal/tunnel/` 真实实现 Local / Remote / Dynamic（SOCKS5）三种模式
>
> 三个特性合并 commit + 单一 tag `v0.6.0`（用户指定，不按惯例拆 v0.6.0/0.6.1/0.6.2）

## 关键设计决策

### A · Multi-hop

1. **x/crypto 锁 v0.31.0** — 保留旧 API（v0.33+ 拆 ssh.NewClient 两步）；用 `client.Dial("tcp", target)` 转发而非手搓 OpenChannel
2. **递归套娃** — `dial(ctx, Target, []Hop)`：最后一跳 = 真实 ssh.Client；中间跳 = `client.Dial` 拿到的 net.Conn 喂给下一跳 `ssh.NewClientConn`
3. **Hop 认证独立** — 每跳可指定不同 method（password / publickey / agent），互不干扰
4. **direct fallback** — v0.5 走 connect.Connector 直拉；v0.6 改成走 `agent.Build("direct", ...)`，对外接口不变
5. **SSHDialer 注入** — main.go 构造 `app.SSHDialer` 包装 `connect.Deps`，给跳板链中每跳复用同一组超时 / 凭据 / host key 校验

### B · Port forward

1. **三种模式共用 client.Dial 路径** — Local / Dynamic 都走 `client.Dial("tcp", target)`；Remote 用 `client.Listen("tcp", bind)`（v0.31 兼容写法）
2. **SOCKS5 极简实现** — Dynamic 模式只支持 no-auth + CONNECT 命令（RFC 1928 最小子集），其他命令 / auth method 一字节 0xFF 拒绝 + close
3. **ClientProvider 接口解耦** — `tunnel.ClientProvider` 让 tunnel 包不直接依赖 session 包（避免循环）；main.go 注入闭包：sessionID → session.Manager.Get → sess.Connector() → `*sshclient.Connector` 断言 → RawClient()
4. **x/crypto v0.31 Listen** — 用 `(*ssh.Client).Listen("tcp", "0.0.0.0:0")`（v0.31 兼容，内部走 `c.SendRequest("tcpip-forward", ...)`），**不**需要 NewClientConn 拆分
5. **状态机** — New → Active → Stopped/Failed；每个 conn 双向 `io.Copy`；Stop 时关 listener + cancel 衍生 ctx

### C · Download streaming

1. **架构对称** — `Download` / `DownloadRequest` / `Downloader` / `DownloaderFactory` 全镜像 upload 那套
2. **断点续传** — manifest 记录已下载 chunk 范围（`[start, end)` 闭区间列表）；重传时跳过
3. **校验和** — 每 chunk sha256，全部完成聚合校验（与 upload 路径一致）
4. **本地文件预分配** — `Truncate(totalSize)` 让"磁盘满"在 Read 之前暴露
5. **进度节流 200ms** — 复用 upload 的 Progress 通道
6. **drag handler / contextmenu 双入口** — frontend 改造为 SFTP 浏览器下载支持

## 踩坑

| 问题 | 修复 |
|---|---|
| C: transfer.Progress 命名冲突 v0.1 stub | rename `Direction` / `EngineProgress`（v0.5.10 同款问题复现） |
| C: chunk sha256 共享 hasher → race | 每 worker 独立 `sha256.New()` |
| C: frontend UploadProgress 组件需扩展方向字段 | 复用同一组件，进度条逻辑不动，加 `direction` prop |
| A: x/crypto v0.31 vs v0.33 API 差异 | 锁 v0.31 写法：`client.Dial` / `client.Listen` 直接可用，不走 NewClient 拆分 |
| A: direct fallback 不能破坏 session.Manager 接口 | 改走 `agent.Build("direct", ...)`，session.Manager 不变 |
| B: tunnel 与 session 循环依赖 | 抽 `ClientProvider` 接口（函数式适配器），main.go 注入闭包 |
| B: Remote forward in-process server 不支持 | 集成测试 mark skip；v0.6.1 补 ssh.NewServer forward handler |
| B: ctx.Done() 在 context.Background() 返回 nil | 已知遗留，v0.6.1 修 |

## 关键 API

```
// A · Multi-hop
agent.Build(ctx, "multi-hop", BuildOptions{Hops: [...], FinalTarget: ...}) (*ssh.Client, error)
agent.Register("direct",       DirectBuildFunc)
agent.Register("single-jump",  SingleJumpBuildFunc)
agent.Register("multi-hop",    MultiHopBuildFunc)

// B · Port forward
tunnel.MemoryManager.WithClientProvider(tunnel.ClientFunc) *MemoryManager
tunnel.MemoryManager.Open(ctx, spec) (Tunnel, error)   // 按 spec.Mode 分派 Local/Remote/Dynamic
tunnel.Tunnel.{Start, Stop, Spec, State, Stats}()

// C · Download streaming
Download(ctx, downloader, req, manifestDir, progress) error
Manager.StartDownload(parentCtx, req) (string, error)
Manager.CancelDownload(transferID) error

// Wails bindings（C）
StartDownload(ctx, req) (string, error)
CancelDownload(ctx, transferID) error
```
