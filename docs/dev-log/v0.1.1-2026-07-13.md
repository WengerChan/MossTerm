## 🔧 修复的 Bug（按发现顺序）

### 1. macOS 沙盒 Documents 目录写不进去 GOMODCACHE
- **症状**：`open /Users/chenwen/Documents/go/pkg/mod/cache/download/...lock: operation not permitted`
- **根因**：`GOMODCACHE` / `GOPATH` 默认指向 `~/Documents/go`，被 macOS 沙盒拦截
- **修复**：export `GOMODCACHE=$HOME/go/pkg/mod` + `GOCACHE=$HOME/go/cache` + `GOPATH=$HOME/go`
- **副作用**：需要每次命令前 export（建议加到 `~/.zshrc`）

### 2. `proxy.golang.org` IPv6 timeout
- **症状**：`dial tcp [2607:f8b0:400e:c1e::8d]:443: i/o timeout`
- **根因**：用户网络对 Google IPv6 不通
- **修复**：`GOPROXY=https://goproxy.cn` 走国内镜像

### 3. `leaanthony/u v1.4.0` 镜像不存在
- **症状**：`reading https://goproxy.cn/github.com/leaanthony/u/@v/v1.4.0.mod: 404 Not Found`
- **根因**：wails 2.9.2 锁的 v1.4.0 在国内镜像没缓存
- **修复**：`go get github.com/wailsapp/wails/v2@v2.12.0` + 手动把 leaanthony/u/slicer/gosod 改到 v2.12.0 用的版本

### 4. embed `frontend/dist: no matching files found`
- **症状**：`pattern frontend/dist: no matching files found`
- **根因**：`//go:embed frontend/dist` 是相对 main.go 所在目录（`cmd/mossterm/`），dist 在项目根
- **修复**：在 `cmd/mossterm/frontend/dist/` 下也放一份 index.html；正式开发用 `wails dev` 它会自动处理

### 5. `argon2.Params` 不存在
- **症状**：`undefined: argon2.Params`
- **根因**：`golang.org/x/crypto/argon2` 包**从来没有**导出 `Params` 类型；agent 写代码时凭印象用了不存在的 API
- **修复**：在 `internal/secret/secret.go` 自己定义 `Params` 结构体（5 个 uint32 字段），`deriveKey` 用 `argon2.IDKey(...)` 调用

### 6. `golang.org/x/crypto` v0.33+ 移除了 `argon2` 包的兼容 alias
- **症状**：升到 v0.33.0 后 `Params` 没了
- **根因**：go mod tidy 主动升级
- **修复**：`go get golang.org/x/crypto@v0.22.0` 锁回老版本

### 7. `internal/config/manager.go` 缺 `log` 字段
- **症状**：`unknown field log in struct literal of type Manager` + `m.log undefined`
- **根因**：Manager struct 在 `config.go` 定义，但 `log *slog.Logger` 字段 agent 漏写
- **修复**：在 `config.go` 加 `log *slog.Logger` 字段 + import `log/slog`

### 8. `internal/session/session_impl.go` 缺 `connect` import
- **症状**：`undefined: connect`（多处）
- **根因**：agent 漏 import
- **修复**：加 `"github.com/mossterm/mossterm/internal/connect"`

### 9. `sync.Once.CompareAndSwap` 在 Go 1.26 不存在
- **症状**：`s.startOnce.CompareAndSwap undefined`
- **根因**：Go 1.22+ 引入 `sync.Once.CompareAndSwap`，但 Go 1.26 又移除了（API 不稳定）
- **修复**：用 `atomic.Bool.CompareAndSwap` 替代，字段名 `started atomic.Bool`

### 10. `agent.NewSSHAgentClient` 不存在
- **症状**：`undefined: agent.NewSSHAgentClient`
- **根因**：agent 用了 x/crypto/ssh/agent 包不存在的函数名
- **修复**：改用 `ssh_agent.NewClient(conn)`，从 `$SSH_AUTH_SOCK` 拿 conn；同时把 import alias 改为 `ssh_agent`，避免与 `agent` 包名冲突

### 11. `ssh.NewClient` API 改了
- **症状**：`assignment mismatch: 2 variables but ssh.NewClient returns 1 value` + `net.Conn does not implement ssh.Conn`
- **根因**：x/crypto v0.33+ 把 `ssh.NewClient` 拆成两步：
  1. `NewClientConn(net.Conn, addr, *ClientConfig) (*ClientConn, <-chan NewChannel, <-chan *Request, error)`
  2. `NewClient(*ClientConn, <-chan NewChannel, <-chan *Request) *Client`
- **修复**：拆成两步

### 12. `ssh.BannerCallback` 返回 `error` 不是 `string`
- **症状**：`cannot use func(message string) string {…} as ssh.BannerCallback value`
- **根因**：agent 凭印象写的签名是 string，实际是 error
- **修复**：wrapper 改 `return nil`

### 13. `*ssh.Session` 不直接实现 `io.ReadWriteCloser`
- **症状**：`*sshSession does not implement connect.Session (missing method Read)`
- **根因**：`ssh.Session` 只有 `StdinPipe()` / `StdoutPipe()` / `StderrPipe()` 三个 pipe，没有直接实现 Read/Write
- **修复**：在 OpenSession 时 `sess.StdinPipe()` + `sess.StdoutPipe()`，存到 sshSession 字段，Read/Write 走 pipe

### 14. `*ssh.Session` 没有 `Pid` 字段/方法
- **症状**：`s.Session.Pid undefined`
- **根因**：x/crypto SSH 协议层就不返回 PID（agent 凭印象写的）
- **修复**：`ShellPID() int { return 0 }` 占位

### 15. `connect.HostKeyCallback` 不能赋给 `ssh.HostKeyCallback`
- **症状**：`cannot use func(...) as ssh.HostKeyCallback value in assignment`
- **根因**：Go 1.26 严格类型系统：named func type 不能隐式转换，即使签名完全一致
- **修复**：把 `connect.HostKeyCallback` 改成 `type HostKeyCallback = ssh.HostKeyCallback`（type alias 而非 new type）

### 16. `internal/connect/factory.go` `net` 未用
- **症状**：`"net" imported and not used`
- **根因**：把 `HostKeyCallback` 改成 type alias 后，`net` 不再被引用
- **修复**：删除 import

### 17. `BannerCallback` 签名也要改 `func(message string) error`
- **症状**：（与 12 同源，但 type alias 同步改）
- **修复**：`type BannerCallback func(message string) error`

## 🎯 关键经验教训

### 1. Go 1.26 严格类型 + 命名 func type
- `type X = ssh.Y`（type alias）比 `type X ssh.Y`（new type）友好得多
- 涉及第三方库回调类型时优先用 alias

### 2. macOS Documents 目录的 GOMODCACHE 限制
- 需要 export `GOMODCACHE=$HOME/go/pkg/mod` 等
- 建议加到 `~/.zshrc` 永久生效

### 3. embed 路径相对 main.go，不是项目根
- `//go:embed frontend/dist` 是相对 main.go 所在目录
- 用 `wails dev` 工具可以处理，开发体验更好

### 4. x/crypto 升级破坏性变化多
- `argon2.Params`：从来没存在过
- `ssh.NewClient` 签名：v0.33 拆两步
- `ssh.BannerCallback`：从返回 string 改为 error（很早期）
- `ssh.Session.Pid`：从未实现
- **建议**：在 go.mod 里用 `// indirect` 锁紧版本，避免自动升级破坏

### 5. SSH 协议层就没暴露 PID
- x/crypto 库的实现没有解析 exec 报文的 pid
- v0.1 用 0 占位是合理选择

### 6. Documents 目录在 macOS 沙盒下要绕开
- 项目可以保留在 `~/Documents/learning/go-prj/MossTerm`
- 但 build 时 cp 到 `/tmp/MossTerm` 跑
- 长期：建议在 `~/go/src/github.com/mossterm/mossterm` 建软链
