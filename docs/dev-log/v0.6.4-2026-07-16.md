# v0.6.4 · 真实 PDF 渲染

> v0.5.9 best-effort 文本片段提取升级为真实渲染：接 pdfjs-dist 6.x，
> 完整 page 渲染到 `<canvas>`，支持翻页 + zoom。

## 关键设计决策

### 1. pdfjs-dist 6.x 选型

- **不用 pdf.js 4.x legacy build**（UMD/global）—— 走 ESM（vite 5.x 原生支持）
- worker 用 `?url` 导入：`import PdfWorker from "pdfjs-dist/build/pdf.worker.min.mjs?url"`
- workerSrc 一次性全局设置：`pdfjs.GlobalWorkerOptions.workerSrc = PdfWorker`
- vite 5.x 自动 chunk：worker 独立 `pdf.worker.min-*.mjs` 1.25MB（不进主 bundle）

### 2. 关键 API 实证

- **PDFDocumentProxy 在 6.x 没有 destroy()** —— 文档句柄由 PDFDocumentLoadingTask 持有，依赖 GC。文档里有注释说明
- **RenderParameters.canvas 是 required**（不再只 canvasContext 即可）
- **page.cleanup() 返回 Promise<void>**（5.x 是 sync boolean）—— await 化

### 3. 路径与大小

- 走整份 PDF（≤50MB）：`SftpReadFileChunk(0, meta.size)` 命中后端 `PREVIEW_MAX_BYTES=50MiB` hard cap，**不用改后端契约**
- >50MB 走 v0.5.9 best-effort 路径（头部 2MB 提文本片段）—— 不渲染
- 新常量 `PREVIEW_PDF_MAX_FULL_BYTES = 50 MiB` 跟后端契约对齐（preview.test.ts "constants match backend contract" 测试覆盖）

### 4. 翻页 race 防护

- `renderTokenRef.current` 每次 render 自增；旧的 render 完成后 check token，不一致就丢弃
- DPR 缩放让 canvas 物理像素跟 CSS 像素对齐（高 DPI 屏不糊）
- zoom 0.25x - 4.0x，步进 0.25

### 5. 资源 / 体积

- wails build 输出 mossterm.app binary **14MB**（v0.6.0 ~12MB，+2MB = pdf.js + worker）
- 前端 bundle：main 549KB（gzip 161KB）+ worker 1.25MB（独立 chunk，按需加载）
- 4.9MB `dist/`（之前 1-2MB）—— 可接受，SFTP 预览不是高频路径

## 已知遗留

- **wails webview worker 路径在 macOS arm64 实机未验**：`new URL(... import.meta.url)` 在 `assets://` 协议下走相对路径——理论上正确，但没在真 app 里打开 PDF 跑过（手动测试留给 v0.6.5+）
- pdfjs-dist 6.x 是当前最新（2026-07-16 校准）；未来小版本若改 API 需重新适配 RenderParameters / GlobalWorkerOptions
- 没做缩略图侧栏 / 文本层点击 / 表单 / 加密 PDF（v0.7+ 排期）
- pdfjs worker 1.25MB 一次性加载；优化路径：dynamic import 触发时再 load（v0.7+）
