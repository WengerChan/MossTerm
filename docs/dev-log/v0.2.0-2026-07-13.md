## 🔧 关键设计决策

### 1. readLoop 双层 goroutine + accumulator

```
[sess.Read]  →  reader goroutine  →  dataCh(cap=8)  →  main loop  →  events(cap=64)
                                                       ↓
                                            accumulator (cap=64KB)
                                            ticker(16ms) 强制 flush
                                            超 64KB 立即 flush
```

| 角色 | 职责 |
|---|---|
| **reader goroutine** | 纯 IO，32KB 读缓冲；`dataCh` cap=8（256KB in-flight） |
| **main loop** | select 协调 `done` / `ticker.C` / `dataCh`；accumulator cap=64KB |
| **flush 触发** | ticker 16ms 一次 / 满 64KB 立刻 / 关停时 best-effort |

### 2. Overflow 处理

- **累计语义**：events 满时整批丢弃，把 `len(ev.Data)` 加到 `overflowBytes`（atomic.Int64），置位 `overflowPending`
- **不累计 state/exit/error**：元事件丢失对终端用户无可见影响
- **fanoutLoop 旁路 emit**：overflow 事件**不**走 events 通道，直接 broadcast 给 subs
- **顺序**：`StorePending(false) → Swap(0) → if bytes>0 broadcast`，并发 Add 时新 bytes 被原子捕获，**v0.2 不丢字节**

### 3. 吞吐预算（写在 `eventsBufferSize` 注释里）

| 场景 | 吞吐 |
|---|---|
| 稳态（每 tick 一次 maxAccum） | 64KB / 16ms ≈ **4 MB/s** |
| 理论上限（events 永不空） | 64 × 64KB / 16ms ≈ **256 MB/s** |
| 典型 SSH 终端 | < 100 KB/s |

超出 4MB/s 触发 overflow 事件——是预期降级路径。

### 4. 关停语义保留
- `Close` → `close(s.done)` → readLoop 走 `<-s.done` 分支 best-effort flush + return
- reader 错误时：推 `readErrCh` + `defer close(dataCh)` → main loop 走 `!ok` 分支最后 flush + emit exit event + `Close`

## 🐛 顺手修的 v0.1.x 残留 bug

`tryPublish` 在关停时**1/3 概率误报 overflow**（v0.1.4 就有）：

```go
// 原代码：
select {
case s.events <- ev:
case <-s.done:
default:
    // 静默丢
}
```

Go runtime 在三个 case 中**均匀随机选**。当 `s.done` 已关闭 + `s.events` 满时，1/3 概率选 default —— 此时是正常关停路径，**不该**被记成数据丢失。

**修复**（写进 `tryPublish` doc comment）：
```go
default:
    if n := len(ev.Data); n > 0 {
        select {
        case <-s.done:
            // 正常关停，不计入 overflow
        default:
            s.overflowBytes.Add(int64(n))
            s.overflowPending.Store(true)
        }
    }
```

**收益**：语义正确，关停时绝不产生幽灵 overflow 事件。`TestTryPublish_SessionClosed_DoesNotIncrement` 是回归测试证据。
