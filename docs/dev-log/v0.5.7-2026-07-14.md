## 关键修复

### 1. `@types/*` 别名冲突（致命，20+ TS6137）
TypeScript 把 `import x from "@types/xxx"` 当成内置 `@types` 包查找，不是路径别名。
**修法**：删掉 tsconfig 里的 `@types/*` 别名，统一用 `@/types/*`（基于现有 `@/*` 别名）。
sed 替换 14 个文件共 20 处 import。

### 2. `terminalStore.ensureConfig` 自引用（TS7022/TS7023）
```ts
// 旧：create<T>() 返回类型推不出，TS 误报 useTerminalStore 是 any
let cfg = useTerminalStore.getState().configs[id];
// 新：用 set 的 get() 闭包，避免在 create 回调里引用 self
set((s) => { cfg = s.configs[id]; ... });
```

### 3. `App.js` 一直不存在（vite build 失败根因）
v0.5.0 之前 `App.d.ts` 是手写占位（`App` class with static methods），
JS 侧 `App.ListProfiles()` 调用全靠它。**`App.js` 从来没存在过**。
`wails build` 不走 vite 所以二进制没影响，但 `npm run build` 一直挂。

**方案**：手写 `App.js` 当 bridge，把 `App.Xxx(args)` 转发到 wails v2.12.0 真实 runtime：
```js
window.go.wailsbindings.App.Xxx(args...)  // wails 自动注入 ctx
```

### 4. `wails build` 找不到 main.go（"no Go files"）
`wails build` 在项目根跑 `go build`（无 path），要 main.go 在根。
**修法**：把 `cmd/mossterm/main.go` 移到 `./main.go`（仍是 `package main`，
但放根就匹配 wails 默认）。`//go:embed frontend/dist` 相对路径自动跟着对。
Makefile 4 处 `./cmd/mossterm` 同步改为 `.`。

### 5. `wails.json` 新增 + `wailsjsdir` 隔离
```json
{
  "wailsjsdir": "./frontend/.wails-bindings"
}
```
让 `wails generate` / `wails build` 自动生成的文件写到 `.wails-bindings/`（git ignored），
不污染手写的 `wailsjs/go/main/`。前端代码一直 import `@wails/go/main/App`。

## 踩坑记录

- **wails v2.12.0 `wails generate module` 输出风格变了**：
  - 旧：`wailsjs/go/main/App.{ts,js}`（手写风格 `App.Xxx()`）
  - 新：`wailsjs/go/wailsbindings/App.{ts,js}`（独立函数 + 显式 `context.Context` arg）
  - 全量迁移前端 = ~30 个 import + App.Xxx → Xxx(ctx, args) 改写，工作量 ≫ v0.5.7 范围
  - **结论**：保留手写 + 写 App.js bridge，wails.json 把 wailsjsdir 隔离到 `.wails-bindings/`
- **`go build -tags bindings` 在根跑需要 main**：
  - 临时方案：`//go:build bindings` stub main.go
  - 永久方案：把真 main 移到根（已做）
- **macOS Documents 沙盒**：v0.5.7 仍需 `cp -R` 到 `/tmp` 才能 build（pre-existing）
