# v0.5.9 · SFTP 文件预览

> **零下载预览**：双击 SFTP 文件弹 PreviewPanel（不替换列表），自动分流
> 图片 / PDF / 文本 / 二进制 / 超大文件 5 种分支。三层大文件保护：后端 hard cap + magic 分类 + 前端阈值。

## 关键设计决策

1. **三层防御 >50MiB**：后端 `ReadFileChunk` 50 MiB cap + `ClassifyPreview` 优先级检查 + 前端 `routePreviewKind` 二次校验
2. **分类优先级**：size cap → magic (PNG/JPEG/GIF/WebP/PDF) → mime (`net/http.DetectContentType`) → 扩展名白名单 → 兜底 binary
3. **PDF 走 best-effort**：不引 pdf.js（~MB 依赖 + 国内镜像问题），用正则提取 header / page count / title / Tj 文本片段
4. **3 binding 语义分工**：`SftpReadFileChunk` 通用读 + `SftpStatFile` lite (mime+size) + `SftpGetFileMetadata` full (magic+classify)
5. **覆盖式 PreviewPanel**：absolute inset-0 z-20 在 SftpBrowserContent 之上，Esc / backdrop / X 三种关闭
6. **零新 npm 依赖**：与 v0.5.8 一致，pure helpers 单测用 `node:test`

## 边界 case

| 场景 | 行为 |
|---|---|
| .jpg 实际是 PNG magic | kind=image（magic 覆盖扩展名） |
| 100MB 文件 | kind=toolarge，前端禁止读字节 |
| 10MB 文本 | 后端自动归类 binary；前端仅元信息 |
| 10MB 图片 | kind=image，但 size > 5 MiB cap → 仅元信息 + 截断提示 |
| 预览期间切 tab | PreviewPanel 仍在，previewPath state 随 SftpBrowserContent 卸载重置 |
| "下载"按钮 | v0.5.9 仅 toast 提示 "v0.6+ 接入"，不实际下载（避免 scope creep） |
