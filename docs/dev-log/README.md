## 命名规范

```
v<MAJOR>.<MINOR>-YYYY-MM-DD.md
```

- `MAJOR` = 大版本（功能阶段）
- `MINOR` = 小版本（迭代）
- `YYYY-MM-DD` = 落盘日期

## 记录模板

每个 dev-log 至少包含：
1. 🎯 目标 —— 本版本要达成什么
2. 📊 数据对比 —— 文件数 / 代码行 / stub 转化为实现的比例
3. 🏗️ 实现总览 —— 改了哪些模块、怎么改、关键点
4. 🔌 接口契约 —— 前端/外部可调用的方法签名
5. 🚀 流程图 —— 启动 / 关键数据流
6. ⚠️ 已知风险 —— 列入 backlog 的妥协
7. 🧪 设计决策 —— 关键架构选择的理由
8. 🛣️ 下一步 —— 候选方向

> 苔藓不急，但它一直在长 —— 每个 dev-log 都是一节新长的"茎"。
