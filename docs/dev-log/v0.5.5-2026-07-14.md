# v0.5.5 · Profile CRUD 接通真实 wails binding

> **v0.1 留的债务清零**：前端 `connectionStore` 5 处 `// TODO: await Api.X()` 注释
> 全部取消，接通真实 wails binding。**typecheck 净 0 增长**（v0.5.4 patch2 50 errors → 49 errors）。

## 改动

| 文件 | 改动 |
|---|---|
| `frontend/wailsjs/go/main/App.d.ts` | + 3 个 binding 类型：`ListProfiles` / `SaveProfile` / `DeleteProfile`（之前 v0.1 时代 stub 没加） |
| `frontend/src/stores/connectionStore.ts` | 取消 5 处 `// TODO` 注释 + 真实调 wails：<br>- `refreshProfiles` → `App.ListProfiles()`<br>- `saveProfile` → `App.SaveProfile(p)`<br>- `deleteProfile` → `App.DeleteProfile(id)`<br>- `refreshSessions` → `App.ListSessions()`<br>- `openSession` → `App.OpenSession({...})`（用 profile 填 host/port/user/auth）<br>- `closeSession` → `App.CloseSession(id, force)` |
| `frontend/src/components/palette/CommandPalette.tsx` | **顺手修 pre-existing 重复声明**：`setActive` 既是 store hook 又是 useState 改名 → `setActiveIdx`（避免 future build 阻塞） |

## 关键决策

- **后端零改动**：`ListProfiles` / `SaveProfile` / `DeleteProfile` 在 v0.1 时期就写在 `wailsbindings/api.go`（line 98-135），跑得通
- **App.d.ts 加 3 行类型**：手写补全 v0.1 时期漏的 stub（profile CRUD 之前没声明）
- **OpenSession 真实路径**：之前 stub 返回 `"stub-session-id"`，v0.5.5 用 profile 字段填 `OpenRequest` 调真实 binding
