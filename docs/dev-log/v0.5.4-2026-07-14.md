# v0.5.4 · CI/CD 修订（ci.yml + release.yml）

> **纯 devops 任务**：让 GitHub workflow 真能 build 出可用的 MossTerm 客户端。
> v0.0 留的 ci.yml 有 6 个问题（GO_VERSION 太老 / 没有 wails build / 没有 release workflow / 等等），
> 全修；新增 release.yml 走 tag push 自动发版。

## 修订前的问题（v0.0 留的 ci.yml）

1. **GO_VERSION=1.22** —— 实际环境 1.26.4，wails 2.12 + x/crypto v0.22 推荐 1.23+
2. **没有 wails build** —— 注释里说"接入 Wails 的 desktop build 见 release.yml（待补）"，但 release.yml 根本不存在
3. **没有 release workflow** —— tag push 不会触发发版，artifact 只在 workflow run 留 30 天
4. **没有 Wails 在 CI 里的安装** + Linux GTK 依赖没装
5. **前端 lint/typecheck 写了 `continue-on-error: true`** 等于没做（应该 fail-fast）
6. **没有 checksum / SBOM** —— release artifact 没有 integrity 验证

## 修订后的两份 workflow

### `.github/workflows/ci.yml`（修订，282 行）

**3 个 job**：

1. **`quality`**（ubuntu-latest，20 min）
   - Checkout
   - Setup Go 1.23 + Node 20
   - `go mod download` / `gofmt` / `go vet` / `golangci-lint` / `govulncheck`
   - 前端：`npm ci` + `npm run typecheck` + `npm run lint` + `npm run build`（**全部 fail-fast**）

2. **`test`**（3 平台 matrix：ubuntu / macos / windows）
   - `go test -race -shuffle=on -coverprofile=coverage.out -covermode=atomic ./...`
   - Ubuntu 上传 coverage 到 Codecov

3. **`build`**（3 平台 matrix：linux/amd64, darwin/universal, windows/amd64，30 min）
   - Linux: 装 `libgtk-3-dev libwebkit2gtk-4.0-dev pkg-config build-essential`
   - `go install github.com/wailsapp/wails/v2/cmd/wails@v2.12.0`
   - `wails build -platform ${{ matrix.platform }} -trimpath -ldflags "..."`
   - 打包成 `mossterm-${{ platform }}.tar.gz / .zip`
   - 上传 `actions/upload-artifact@v4`

**LDFLAGS**（与 Makefile 同款）：
```
-s -w \
  -X 'github.com/mossterm/mossterm/internal/app.Version=$version' \
  -X 'github.com/mossterm/mossterm/internal/app.Commit=$commit' \
  -X 'github.com/mossterm/mossterm/internal/app.BuildDate=$build_date'
```

### `.github/workflows/release.yml`（新增，214 行）

**触发**：
- `push: tags: ['v*']` —— tag push 自动发版
- `workflow_dispatch` —— 手动指定 tag（dr 验证 / 补发）

**2 个 job**：

1. **`build`**（3 平台 matrix，跟 ci.yml 一致，30 min）
   - 同样装 Wails + 平台依赖 + 跑 `wails build`
   - 解析 tag 版本（push 触发用 `github.ref_name`，dr 触发用 `inputs.tag_name`）
   - 上传 artifact（命名 `mossterm-${{ platform }}`）

2. **`release`**（ubuntu-latest，10 min，needs: build）
   - 下载所有 build artifact 到 `dist/`
   - 生成 `sha256sum mossterm-* > checksums.txt`
   - `gh release create $TAG ./dist/* --title "$TAG" --generate-notes --target $SHA`
   - Release 已存在则跳过（dr 重跑不覆盖）

**关键设计**：

- **不发 pre-release**（v* 默认 stable；alpha/beta 走 dr 也不标 pre）
- **不发版与 ci.yml 的 build 分离**：发版要锁死的 LDFLAGS + 干净 tree + 可重复；ci.yml build 失败不会污染 release
- **concurrency group = `release-${{ ref_name }}`**：同一 tag 不并发（dr 重跑时旧的 cancel）
- **`permissions: contents: write`**：创建 GitHub Release 需要

## 已知风险 / 后续优化

1. **darwin/universal build 在 macOS-latest runner 上**：依赖 lipo（Xcode CLT 自带）。
   偶尔 universal build 会失败，回退方案是拆成 darwin/amd64 + darwin/arm64 两个 matrix
2. **Linux build 需要 webkit2gtk-4.0**：如果 GitHub Ubuntu image 升到 24.04，可能要
   改 `libwebkit2gtk-4.1-dev`（v0.5.4 时用 4.0 稳定）
3. **Windows build 需要 WebView2 Runtime**：Windows-latest 自带；老 Windows
   用户需要手动装
4. **macOS code signing / 公证没做**：GH 出来的 .app 没签名；用户首次打开会被
   Gatekeeper 拦截（要右键"打开"）。v0.6+ 加 `codesign` + `notarytool` workflow
5. **没有 SBOM**：`govulncheck` 是漏洞扫描，不是 SBOM 生成。v0.6+ 加
   `anchore/sbom-action` 出 SPDX SBOM
6. **没有 `wails build -clean`**：当前 build 不强制清空 dist/，可能用到旧产物
   引发神秘 bug。后续加 `-clean` flag
