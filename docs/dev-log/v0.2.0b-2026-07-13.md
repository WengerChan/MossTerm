## 🏗️ 走的方案：B（自实现），不用 x/crypto 标准库

### 为什么不是 A

按你给的提示先去 `ls + grep` 实际确认了 `x/crypto/ssh/knownhosts` 的导出 API：

```bash
$ grep -E "^type|^func" knownhosts.go
type addr struct{ host, port string }                 # 私有
func (a *addr) String() string { ... }
type matcher interface { match(addr) bool }           # 私有
type hostPattern struct { ... }                        # 私有
type hostPatterns []hostPattern                        # 私有
type keyDBLine struct { ... }                          # 私有
type hostKeyDB struct { ... }                          # 私有
func newHostKeyDB() *hostKeyDB                         # 私有
func (db *hostKeyDB) IsHostAuthority(...) bool         # 方法
func (db *hostKeyDB) IsRevoked(key *ssh.Certificate) bool
func (db *hostKeyDB) check(address string, ...) error
func (db *hostKeyDB) parseLine(line []byte, ...) error
func newHostnameMatcher(pattern string) (matcher, error)  # 私有

type KnownKey struct { ... }                           # 导出
type KeyError struct { ... }                            # 导出
type RevokedError struct { ... }                        # 导出

func New(files ...string) (ssh.HostKeyCallback, error)  # 唯一构造函数
func Normalize(address string) string                   # 导出
func Line(addresses []string, key ssh.PublicKey) string # 导出
func HashHostname(hostname string) string               # 导出
```

**结论**：
- `hostKeyDB` / `hostPattern` / `newHostKeyDB` / `newHostnameMatcher` 全部小写，**不导出**
- 唯一构造函数 `New` 返回 `ssh.HostKeyCallback`（不是 DB）
- 你的 "NewDB" 假设**不成立**

### 走 B 的原因

- 我们要"首次连接自动信任 + 持久化"，需要 callback 命中"未找到"分支时手动 Add
- 标准库的 callback 在未知 host 时直接返回 error（不允许写回），不满足需求
- 选 B 自实现，但**匹配算法 1:1 port 自 OpenSSH `addrmatch.c`**（与标准库 `wildcardMatch` 等价）

## 🎯 关键设计决策

### 1. 数据结构：从 map 改为 slice

**v0.1.3**：`map[host]map[keyType]keyBase64` — 精确 host 查表
**v0.2.0b**：`[]entry`，每行 entry = `[]pattern + ssh.PublicKey` — 支持多 host + 模糊匹配

```go
type entry struct {
    patterns []pattern           // 同一行逗号分隔的多 host
    key      ssh.PublicKey
    keyType  string              // 冗余保留（写回文件时用）
}

type pattern struct {
    host string                  // 支持 * / ? 通配符
    port string                  // "22" = OpenSSH 默认
}
```

**为什么改**：通配符 / IP 范围无法用 host 字面值做 map key。必须线性扫描 + wildcard 匹配。

### 2. 匹配算法 = OpenSSH addrmatch.c 语义

**`wildcardMatch`** 直接 port 自 x/crypto/ssh/knownhosts（已与 OpenSSH `addrmatch.c` 对齐）：

```go
func wildcardMatch(pat, str string) bool {
    // 规则：
    //   '*' 匹配任意字符序列（含空）
    //   '?' 匹配任意单字符
    //   其他字面相等
    // 与 filesystem glob 不同：'*' 跨 '.' 也匹配
    //   foo.example.com 是 *.example.com 的有效匹配 ✓
    //   *.example.com 不匹配 example.com（OpenSSH 规则）✓
}
```

**`pattern.match(addr)`**：

```go
// 1. 端口必须完全相等（OpenSSH 规则）
// 2. host 部分走 wildcardMatch
```

**关键不变量**：
- `*.example.com` **不**匹配 `example.com`（基域）
- `192.168.1.*` 匹配 `192.168.1.1` 但**不**匹配 `192.168.2.1`
- `*` 匹配任何 host（含 `""` 空字符串）

### 3. 端口感知：pattern.port + addr.port 都参与匹配

| 文件 pattern | 实际连接（callback host） | 结果 |
|---|---|---|
| `example.com` | `example.com` | ✅ 匹配（port 22 = 22） |
| `example.com` | `example.com:2222` | ❌ 不匹配（22 ≠ 2222） |
| `[example.com]:2222` | `example.com:2222` | ✅ 匹配 |
| `[example.com]:2222` | `example.com` | ❌ 不匹配（22 ≠ 2222） |
| `[::1]:2222` | `[::1]:2222` | ✅ 匹配（SplitHostPort 去括号） |

**关键决策**：端口不匹配 = 不同 host，**不**走 auto-trust 写新 entry（除非回调路径 — 那里仍写）。这是 OpenSSH 语义。

**`loadFromFile`**：`parsePattern` 用 `net.SplitHostPort` 拆
- `[host]:port` → host + port
- `host:port` → host + port
- `host`（无端口）→ host + "22"（默认）

**`HostKeyCallback`**：`parseAddr` 同样拆
- SSH client 传 `host` 参数优先
- 解析失败时回退到 `remote`（x/crypto 的同样兜底）
- 都解析失败 → 返回 error（**不放行** — 避免对未知 host 静默接受任意 key）

### 4. 多 host 字段：每行拆分独立 pattern

```text
example.com,192.168.1.10,*.internal ssh-ed25519 AAAA
```

v0.1.3：只取第一个 `example.com`，其余 2 个 host 静默丢失
v0.2.0b：3 个独立 pattern 共享同一 key，任一匹配 + key 匹配 = 放行

**MITM 检测**：任一 pattern 命中但 key 不匹配 = `ErrHostKeyMismatch`（与 v0.1.3 行为一致）

### 5. HASH 编码 `|1|salt|hash`：识别后跳过

按你 spec："不实现，只识别后跳过"。两种跳过：
- 行首为 `|` → 整行跳过（与 x/crypto 的 `parseLine` 行为对齐）
- 同一行多 pattern 中有 `|...` → 整行返回 error → 跳过（保守：避免一个 host 有 hash 另一个没有时匹配错乱）

**已知妥协**：hash 编码的 host 永远不能匹配 — 那些"已知 host"被 MossTerm 视为"未知"。**v0.2 后续**：可以从 x/crypto 搬 `newHashedHost` + `hashHost` + `decodeHash` 实现完整 hash 匹配。

### 6. `formatPattern` 写回：port 非默认时强制方括号

```go
func formatPattern(p pattern) string {
    if p.port != "22"    { return "[" + p.host + "]:" + p.port }
    if strings.Contains(p.host, ":") { return "[" + p.host + "]" }  // IPv6 + 默认 port
    return p.host
}
```

**与 OpenSSH `Normalize` 对齐**：非默认端口必须加方括号（即使 IPv4），保证其他 SSH 工具读这个文件也能正确解析。

### 7. callback 入参 host 解析失败 = 拒绝

```go
if a.host == "" {
    return fmt.Errorf("knownhosts: cannot determine host from address %q", host)
}
```

**v0.1.3 行为**：return nil（放行） — **这是 bug**：对未知 host 静默接受任意 key
**v0.2.0b 行为**：return error — 与 x/crypto 行为一致，让 SSH 握手失败而不是默默接受 MITM

`TestHostKeyCallback_RejectsUnparseableHost` 是回归测试证据。

### 8. `Add` 入参校验

```go
parsePattern(s) → 空字符串 / 含逗号 → error
```

- 空字符串 → 避免写出 `" ssh-ed25519 ..."` 垃圾行
- 含逗号 → 强制单 host（多 host 形式只在 file 中支持；Add API 不暴露）

`TestAdd_RejectsEmptyHost` / `TestAdd_RejectsMultiHost` 覆盖。

## 🔄 向后兼容

### 文件格式不变

- 旧 known_hosts 文件（v0.1.3 写的）一行 = 一个 entry
- 新代码 parse 旧行：pattern 解析为 `{host: host, port: "22"}` + key 验证
- 行为：`example.com` (port 22 默认) 的旧条目，匹配 `example.com:22` 连接的 callback — **与 v0.1.3 行为一致**

### 公共 API 完全不变

外部 4 个调用点（`cmd/mossterm/main.go`、`internal/sshclient/client.go`、`internal/connect/factory.go`、`internal/session/manager.go`）全部用：
- `knownhosts.New(path)`
- `*knownhosts.Manager`
- `Manager.HostKeyCallback()`
- `Manager.Close()`

**全部签名 + 语义不变**。无需改任何调用方。

### 行为变化的 3 个地方（都是更严格、更安全）

| 场景 | v0.1.3 | v0.2.0b | 风险 |
|---|---|---|---|
| `host,foo.com`（多 host 文件行） | 只取 host | 取 host + foo.com | 行为增强：以前不被记住的 host 现在能匹配 |
| 空 host 回调 | return nil（放行） | return error（拒绝） | 更安全：避免 MITM |
| 端口不匹配的连接 | 直接 auto-trust 写新 entry | auto-trust 写新 entry（**行为一致**） | 端口 22 vs 2222 仍视为不同 host（OpenSSH 语义） |

第 1 个是**增强**（用户显式批准），第 2 个是**修复 bug**，第 3 个是**OpenSSH 对齐**。

### 数据结构迁移

旧 `m.keys map[string]map[string]string` → 新 `m.entries []entry`
- 旧结构在 `New` 时从文件构建，**进程内状态无持久化**
- 升级不需要迁移：旧文件直接被新代码解析
- `Size()` 语义微变：从"distinct host 数"改为"entry 数（=文件行数）"；无外部调用方使用
