模块划分

16 Go packages · 4 Frontend layers
cmd/mossterm
主入口。解析 CLI flag、构造 App、启动 Wails 窗口、信号处理优雅关闭。
main.go · ~95 lines
internal/app
应用生命周期。聚合所有 Manager、响应 OnStartup / OnDomReady / OnShutdown 钩子。
app.go
internal/session
业务层会话。状态机：Connecting → Authenticating → Established → Closing → Closed → Failed。三个 goroutine：connect / read / write。
session.go · manager.go
internal/connect
协议无关的连接抽象。Connector 接口 + AuthMethod sum-type + 全局 Registry 注册中心。
connector.go · factory.go · registry.go
internal/sshclient
SSH 客户端。基于 golang.org/x/crypto/ssh，支持 password / publickey / agent / keyboard-interactive。
client.go
internal/sftpclient
SFTP 客户端。基于 pkg/sftp，封装 list / upload / download / mkdir / rename。
client.go
internal/pty
PTY 封装。基于 creack/pty，跨平台抽象，Resize / SetWinSize。
pty.go
internal/config
配置管理。TOML 格式、热加载、profile 树、默认值合并。
config.go · loader.go
internal/secret
凭据存储。系统 keyring 集成（macOS Keychain / Windows Cred Mgr / Linux Secret Service）。
secret.go · keyring.go
internal/transfer
文件传输引擎。多连接并发、限速、断点续传、校验。
transfer.go
internal/tunnel
端口转发。本地 / 远程 / 动态（SOCKS），独立 goroutine 池。
tunnel.go
internal/agent
跳板链。ChainProxy / 多跳代理、连接复用。
agent.go
internal/plugin
插件骨架。WASM 沙箱（wazero）、权限模型、生命周期管理。
plugin.go
internal/ai
AI 辅助。命令解释、日志总结、批量命令生成、Provider 抽象。
ai.go
internal/ui/wailsbindings
Wails 绑定层。把 Go 方法暴露成前端可调用的异步 API。
api.go
pkg/mossterm
公共 API。供外部工具/插件引用的稳定入口。
mossterm.go

下一步

From skeleton to v0.1
1 · 安装环境
安装 Go 1.22+、Node 18+、Wails CLI 2.x。
./scripts/install.sh 一键搞定（macOS / Linux），Windows 跑 .\scripts\install.ps1。
2 · 启动开发
make wailsdev 启动 Wails dev 模式。
或 make dev 同时跑 Go 后端 + Vite 前端。
3 · 实现 SSH 客户端
把 internal/sshclient/client.go 从 stub 变成真实现。
基于 golang.org/x/crypto/ssh，参考架构文档第 6 章接口契约。
4 · 联通 Wails 绑定
实现 internal/ui/wailsbindings/api.go 的方法，
让前端能调 OpenSession / SendInput / Resize / Close。
