# v0.5.0 A · SFTP 客户端子任务交付

**派活时间**：2026-07-14  
**执行者**：sub-agent (coder)  
**目标**：把 `internal/sftpclient` 8 个 stub 替换为真实 `github.com/pkg/sftp` 实现

## 范围

8 个方法：Open / List / Stat / ReadDir / Open(file) / Read / Write / Close
+ 辅助：Remove / Rename / Mkdir / Rmdir

## 关键决策

1. **List 分页简化**：`pkg/sftp.ReadDir` 不支持分页，v0.5.0 一次性返回全量；
   `pageSize/NextToken` 接口参数保留占位（v0.5.1+ 接真实分页协议时签名不变）
2. **Close 后 nil-safe**：所有方法在 `c.sc == nil` 时返回 `errors.New("client closed")`，
   不 panic（关键：避免 SSH 断连后前端调到残留 client 引发 UI 崩溃）
3. **不防御 nil `*Client`**：让 Go 自然 nil deref 暴露 bug，而非 silently swallow
4. **不加互斥锁**：pkg/sftp 自己的 client 是 goroutine unsafe，加锁责任在调用方
5. **ctx 参数保留但不用**：SFTP 协议无 cancel 机制；保留为对齐上层 ctx-aware 约定

## 教训

- pkg/sftp 的 API 比 ssh 简单很多（一个 SFTP subsystem 而已），但 **goroutine unsafe**
  的设计需要上层包装时心里有数
- interface 占位（pageSize/NextToken）是好习惯 —— 即使当前实现 no-op，也比"现在不接
  以后改签名"省事
