## 关键设计决策

### 1. Pane 类型扩展（kind 字段）
- 旧：`split: null | 'horizontal' | 'vertical'`，leaf vs split 靠 `split === null` 区分
- 新：加 `kind: 'terminal' | 'sftp' | 'split'`，**类型显式**，避免 split 节点误判为 leaf
- `split: null` 仍保留（leaf 节点的 split 必为 null，类型约束双重保险）

### 2. 算法层抽离（paneTree.ts）
- 把 v0.5.0 B 写在 tabsStore.ts 里的 `splitPaneInTree` / `closePaneInTree` /
  `findFirstLeafId` 抽到 `paneTree.ts`（pure functions，无 React/zustand 依赖）
- tabsStore 瘦身：从 ~330 行到 ~310 行（拆算法 + 加重导出）
- **附带修了一个 v0.5.0 B 的 bug**：嵌套 split 关闭 + children lift 时
  `closePaneFromTree` 把外层 split 节点保留为 `children: [l1, l2, l3]`（3 个 child），
  违反"split 节点 = 2 child"软约束。修复：当 `sub.length > 2` 时统一平铺到 root 层。
- **createSplit 修复**：原实现复用 `left.id`（注释留 "调用方负责 id 重新分配"），
  v0.5.8 改为始终分配新 id（接受可选 `idFactory` 入参），消除"唯一根 = target"误判。
  当前无 caller 依赖旧行为，无破坏性。

### 3. SftpBrowserContent 抽离
- v0.5.1 的 SftpBrowser 是个 ~700 行大组件（业务 + modal 包装 + 子 dialog 混在一起）
- v0.5.8 拆出 **业务层** `SftpBrowserContent`（list + toolbar + drag + dialog + viewer）
  + 两个 wrapper：
  - `SftpBrowser`（modal 包装：title + X 按钮 + fixed inset-0 backdrop）
  - `SftpPaneView`（pane 包装：mini header + EmptyLeafHint + focus ring）
- 共享 SftpBrowserContent → 0 行业务代码重复

### 4. addSftpPane 语义
- 不做自动 wrap（"root 唯一 terminal 时自动横拆"）—— 太隐式
- 简单平铺到 root：`addPaneToRoot(panes, leaf)` 直接 append
- 后续用户可手动 split 调整布局
- 缺省 `sessionId`：用 `tab.sessionId`（tab 创建好 session 后再调即可）

### 5. Tab title pane 概要徽标
- 多个 terminal 时显示 "term ×N" 后缀
- 有 SFTP 时显示 📁 SFTP 图标（v0.5.8 badge）
- 单 terminal + 无 SFTP：不显示（保持 tab 简洁）

### 6. App.d.ts 不动
- 后端 binding 早已有 SFTP 系列（v0.5.1）
- v0.5.8 只改前端 UI 集成，零后端改动
- App.d.ts stub 已声明 `SftpList/Stat/Mkdir/Remove/Rename/Read/Write/UploadFile`

### 7. 测试策略
- **不加 vitest 依赖**（保持 v0.5.7 baseline npm 依赖 0 变化）
- 用 Node 22+ 内置 `node:test` + `--experimental-strip-types`（Node 26 GA）
- 测 paneTree.ts 算法的 22 个 case（pure functions，无需 React/zustand）
- `tabsStore.test.ts` 因级联 import 需要 `.ts` 后缀（Node strip-types 严格），
  与 Vite 项目 ESM 解析不兼容，**放弃 store-level 集成测试**（store actions 太薄，
  算法已在 paneTree 测覆盖）

### 8. SftpPanel 旧 stub 保留
- `SftpPanel.tsx`（v0.1 占位）和 `sftpStore.ts`（v0.1 stub）保留不动
- `MainLayout` 的"sftpPanelVisible"路径（Ctrl+Alt+S 切换）保留

## 边界 case

| 场景 | 行为 |
|---|---|
| tab 没 session 就 addSftpPane | 新 SFTP leaf `sessionId = null`，渲染 EmptyLeafHint 引导用户先连 SSH |
| addSftpPane 多次 | 多个独立 SFTP leaf（每个独立 path 状态，符合 v0.5.1 用户预期） |
| 关闭 SFTP leaf | pane 树正确 collapse / lift；active 兜底到剩余 leaf |
| 关闭唯一 terminal leaf | 整 tab 关闭 |
| split 内关 nested split 节点 | **修复 v0.5.0 B bug**：children 全部 lift 到 root 层（不再生成 3+ child 的 split 节点） |
| tab 含 SFTP → TabBar SFTP 按钮 | 仍可加（"再开一个 SFTP"）；按钮提示文案更新为 "在当前 tab 加 SFTP pane" |
| session:exit 关闭 SSH session | SFTP leaf sessionId 仍在；后续 SftpList 会失败，UI 显示错误（pane 自身不自动关闭） |
| SplitPane 工具栏 close 关闭 split 节点 | children lift 到上一层（v0.5.0 B 行为保持） |
| Esc 在 SFTP pane 内 | 关闭内部 viewer/dialog，不关 pane（pane 关闭走 SplitPane 工具栏） |
| 已开启的 SFTP pane 被切到非 active | focus 仍保留在原 active pane；切回时 LeafPane useEffect 重新 focus |
