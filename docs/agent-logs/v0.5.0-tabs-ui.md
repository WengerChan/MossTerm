## 关键决策

1. **Pane 树 = 二叉树**：节点要么是 Split（带 direction + 两棵子树）要么是 Leaf（带 sessionId）
   - 不用 N-ary 拆分：UI 操作（方向键 / drag）天然对应二叉
   - 不做 pane 拖拽换位：v0.5.0 范围之外
2. **splitPane(tabId, paneId, direction, newSessionId)**：插入 Split 节点，
   原 sessionId 保留到第一个子节点，新 session 放第二个 —— 保留用户当前焦点
3. **closePane(tabId, paneId)**：删除 pane，若父 Split 只有一个孩子则"上提"（collapse），
   若 Split 变空则连父 Split 一起删 —— 避免留空 Split 节点
4. **Terminal mount guard**：`sessionId !== null` 才挂载 xterm.js —— 避免空 pane 渲染崩溃
5. **Zustand store**：tabs/panes 树 + active 状态集中管理，避免 React 状态分散
