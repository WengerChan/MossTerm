## 关键设计决策

### 1. 三层防御：大文件保护
- **后端 hard cap**：`PreviewMaxBytes = 50 << 20`（50 MiB），`ReadFileChunk` 自动截断
- **后端分类**：`ClassifyPreview` 第一步就检查 `size > PreviewMaxBytes` → "toolarge"
- **前端 belt-and-suspenders**：`routePreviewKind` 同样检查 `size > PREVIEW_MAX_BYTES` → "toolarge"
- **文本阈值**：`PreviewTextMaxBytes = 5 MiB`，超阈值按 binary 处理
- **图片 cap**：`PREVIEW_IMAGE_MAX_BYTES = 5 MiB`，base64 内联上限

### 2. 分类优先级
1. `size > 50 MiB` → toolarge
2. magic (PNG/JPEG/GIF/WebP/PDF) → image / pdf（覆盖错误扩展名）
3. mime（`net/http.DetectContentType`，前 512 字节）
4. 扩展名白名单（md/txt/go/py/json/yaml/...）
5. 兜底 binary
6. 兜底升级：kind=binary + 无 mime + 无 ext + head 无 NUL → text

### 3. 三个 binding 的语义分工
- `SftpReadFileChunk`：通用 read，offset+size 任意，50 MiB cap
- `SftpStatFile`：lite（仅 mime + size，不做 magic/classify）
- `SftpGetFileMetadata`：full（magic + classify，返回 Kind）
- 前端只用 1 + 3；2 是 spec 明确要求的"轻量"入口（未来 tooltip 等用）

### 4. PDF 预览：best-effort 而非完整渲染
- **不引 pdf.js**（spec 明确：~MB 依赖 + 国内镜像问题）
- 实现：`extractPdfTextSnippet` 用正则提取：
  - PDF header (`%PDF-x.y`)
  - Page count (`/Count N`)
  - Title/Author/Subject（metadata dictionary，括号 + hex UTF-16 BE 两种格式）
  - Content stream 的 `Tj` 文本操作符（最多 30 个片段）
- UI 明确标注 "best-effort，完整渲染留 v0.6+" —— 诚实
- 如果 v0.6+ 接 pdf.js，可平滑替换 `PdfView` 子组件

### 5. PreviewPanel 设计
- 覆盖在 SftpBrowserContent 上（absolute inset-0 z-20），不替换列表
- Esc / backdrop / X 三种关闭方式
- 加载流程：先 metadata → 按 kind 二次拉字节（避免无谓的读）
- 文本分支提供 "text / hex" 切换
- "下载"按钮：v0.5.9 仅 toast 提示 "v0.6+ 接入"，不实际下载（避免 scope creep）
- "复制路径"按钮：实际可用（navigator.clipboard）

### 6. 大文件保护对 SftpRead/SftpWrite 的影响
- **未改** `SftpRead`：仍是 1 MiB hard cap，限定小文件编辑器
- **未改** `SftpWrite`：同 SftpRead
- 新加 `SftpReadFileChunk`：50 MiB cap，offset 任意
- 三个 binding 互补，前端按需选用

### 7. SftpBrowserContent 的清理
- 删 `TextViewer` 子组件（~60 行）
- 删 `looksLikeText` 启发式（移到 PreviewPanel 的 `textContent` 加载路径）
- 删 `onViewerOpenChange` prop（v0.5.8 预留，v0.5.9 无 caller）
- 删 `TEXT_PROBE_BYTES` / `TEXT_PRINTABLE_THRESHOLD` / `TEXT_PREVIEW_MAX_BYTES` 常量
- 删 `X` import（仅 TextViewer 用）

### 8. App.d.ts 镜像
- v0.5.9 加 `PreviewMetadata` interface（与 backend struct 镜像）
- `kind` 字段用 union type 约束：`"image" | "pdf" | "text" | "binary" | "toolarge" | ""`
- `mode` 字段标注 uint32 → number（Wails 序列化规则）
- `modTime` 字段标注 RFC3339 字符串（Go time.Time 序列化规则）

### 9. 测试策略
- **后端**：preview.go 28 case（pure functions + boundary）
  - `DetectMagic`：PNG/JPEG/GIF/GIF87a/WebP/PDF + unknown 边界
  - `ClassifyPreview`：all path combinations（toolarge/magic/mime/ext/fallback）
  - `isTextExt`：whitelist/non-whitelist
  - `looksLikeTextBytes`：empty/utf8/null/binary
  - `ReadFileChunk`：client closed + threshold 验证
- **前端**：preview.test.ts 28 case（pure functions，node:test）
  - `routePreviewKind`：all 5 kinds + size 兜底
  - `uint8ArrayToBase64`：empty/round-trip/非 ASCII/100KiB 大数据
  - `hexDump`：empty/single line/multi line/ASCII/不可打印/截断
  - `extractPdfTextSnippet`：header/page count/title/Tj/escape
  - 常量稳定性测试（防止 silent 改动破坏后端契约）
- **未做** `PreviewPanel.tsx` 组件测试（React component，需要 vitest/jsdom —— 与 v0.5.8 一致保持 0 npm 依赖）

## 边界 case

| 场景 | 行为 |
|---|---|
| 双击目录 | `cd` 进入（v0.5.1 行为不变） |
| 双击 .jpg 文件（实际是 PNG magic） | kind=image（magic 覆盖扩展名） |
| 双击 .bin 文件（实际是 PDF magic） | kind=pdf（magic 覆盖扩展名） |
| 100 MB 文件 | kind=toolarge，前端禁止读字节 |
| 10 MB 文本 | kind=（后端自动）binary；前端仅显示元信息 |
| 10 MB 图片 | kind=image，但 size > 5 MiB cap → 仅显示元信息 + 截断提示 |
| 文件不存在 | SftpGetFileMetadata error → ErrorView |
| session 断开 | SftpGetFileMetadata error → ErrorView |
| 预览期间切到其它 tab/pane | PreviewPanel 仍在，但 SftpBrowserContent 卸载时 previewPath state 随之重置 |
| Esc 在 SFTP pane | 关闭 PreviewPanel，不关 pane（v0.5.8 行为） |
| "下载"按钮 | v0.5.9 toast "v0.6+ 接入"；不实际下载 |
| "复制路径"按钮 | 实际复制 + success toast |
| 文本分支"hex" 模式 | xxd 风格 16 字节/行，最大 16 KiB（截断提示） |
| PDF 分支 | 提取 header / page count / title / author / Tj 文本片段；不渲染页面 |

主理后续建议（不在 v0.5.9 范围）：
1. **真实 PDF 渲染**：v0.6+ 接 pdf.js（dist bundle 约 1MB，国内镜像可能没缓存；或自实现 spec 解析）
2. **真实下载按钮**：v0.6+ 接 Wails SaveFileDialog + SFTP streaming upload/download
3. **图片缩略图缓存**：同一文件多次预览时复用 base64（v0.6+ LRU 缓存）
4. **二进制预览**：head 字节 hex dump + 选中的偏移跳转（仿 hex editor）
5. **预览历史**：用户最近预览过的 N 个文件（v0.6+ 持久化到 config）
6. **PreviewPanel 单元测试**：加 vitest 后补（v0.5.8 决定不加 npm 依赖）
7. **PDF 文本片段提取增强**：支持 `<hex> Tj`（hex 字符串格式）、`TJ` 数组格式、`'` 操作符
8. **预览面板宽高记忆**：用户拖拽后记住 size（per-leaf 状态，v0.6+）
