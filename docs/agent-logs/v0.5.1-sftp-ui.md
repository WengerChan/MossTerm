## 关键决策

1. **Modal 而非 Drawer**：看 MainLayout 现状（侧栏 + 中部 Tab/Pane + 右侧 SftpPanel stub），
   再加一个右侧 drawer 会跟既有 SftpPanel 视觉/状态冲突；独立 Modal 跟 TrustRequestModal /
   CommandPalette 风格一致，z-index 凌驾于普通 UI 之上
2. **子对话框不依赖 uiStore.modal**：mkdir/rename/delete/查看四个子对话框直接渲染在
   SftpBrowser 内部 `absolute inset-0` overlay。理由：uiStore 是单 modal slot，多个子对话框
   会互相冲掉；本地 state 简单清晰，esc 关闭顺序（dialog → viewer → browser）也好控制
3. **单实例 SFTP 浏览器**：v0.5.1 不支持"每个 tab 开一个浏览器"。`sftpBrowserStore.open(sid)`
   直接覆盖 `sessionID`，用户体感 = "切到那个 session 的目录"（v0.6+ 接 Pane 树再考虑多实例）
4. **文本探测启发式**：NUL 字节 → 二进制；否则用 `TextDecoder('utf-8', { fatal: true })` 试解码；
   解码成功再看可打印字符比例（≥ 85%）。NUL 字节门控比 BOM 嗅探简单可靠，避免误判 UTF-16
5. **stale response 防护**：每次 `App.SftpList` 调 `++reqTokenRef.current`，回调时校验
   当前 token；防止 cd → cd 快速切换时旧响应覆盖新结果
6. **关闭浏览器 = 清状态**：useEffect 监听 `open` 变化，关闭时重置 path/entries/dialog/viewer。
   下次打开从根目录开始（不保留上次浏览位置）
7. **sessionID 重置 effect**：sessionID 变化时同样重置（store 切换 session 时不会留上次
   的目录缓存）。两个 effect 看似重复，但职责不同 —— 一个管"开关"，一个管"换 session"
8. **Wails binding 选择方案 B（扩展 App.d.ts）而非 shim**：`App.d.ts` 已经在 v0.5.0
   时存在，扩展它更接近 Wails 真实生成结果，主理跑 `wails generate` 时只会被合并
   而非删除。shim 方案（`frontend/src/wails/sftp.d.ts`）保留为备选
9. **不接 sftpStore 缓存**：v0.5.1 每次 List 都打后端，不复用 `useSftpStore.entriesByPath`。
   理由：sftpStore 用的旧 `SftpEntry` 类型（带 `type: 'file'|'dir'|...`）和现在的
   `Entry` 类型（`isDir` / `isSymlink`）字段不同；混用会引入 type adapter，得不偿失。
   v0.5.1 SftpBrowser 自管 entries 状态

### SftpBrowser 组件结构

- **SftpBrowser（主组件）** —— 顶层 fixed 容器，920×640 Modal
  - Header：title + host label + X
  - Toolbar：上/刷/新建 + Breadcrumb + 错误提示
  - Column header：可点击排序（name/size/modTime）
  - Entry list：双击 = 进目录/查看文件，单击 = 选中
  - Action bar：选中后的查看/重命名/删除
  - 嵌套 SubDialog（mkdir/rename/delete）
  - 嵌套 TextViewer（文本/二进制）
- **Breadcrumb** —— 段可点，根用 `/` 表示
- **SortHeader** —— 复用同一组件，按列点击切 asc/desc
- **EntryRow** —— 名称（目录加粗 + 图标）/ 大小 / 时间 / 权限（rwxr-xr-x）
- **SubDialog** —— 三态（mkdir 输入 / rename 输入预填 / delete 确认）
- **TextViewer** —— 文本（截断 256 KiB）/ 二进制（"binary file, N bytes"）
- **本地 formatters** —— `formatModTime`（RFC3339 → "YYYY-MM-DD HH:mm"）、
  `formatPerms`（mode → "rwxr-xr-x"）。mode 取低 12 位（`& 0o7777`）以兼容 setuid/setgid/sticky 位

## 已知限制（v0.5.1 接受，v0.6+ 优化）

- **不做上传/下载**：任务范围之外；`SftpWrite` / `SftpRead` 的写路径留给 v0.6+
- **不做拖拽 / 大文件分片 / 进度条 / 搜索 / 多选 / 真实分页**：都列在任务"不做"里
- **不做 session 关 → 自动关浏览器**：用户在 modal 里操作时若 session 断了，
  会看到 toast 报错但 modal 不会自动关；v0.6+ 加订阅 session:state 事件联动
- **二进制文件只显示 "binary file, N bytes"**：连 base64 预览 / hex dump 都不做
  （v0.6+ 再说，hex dump 体验不好）
- **Path helpers 简单实现**：`normalizePath` / `joinPath` / `parentPath` 是手写 30 行
  工具函数；v0.6+ 拆到 `@utils/path` 公共模块
- **TypeScript strict 兼容**：`useConnectionStore((s) => s.sessions[sessionID])` 没
  用 `useShallow`，理论上任何 store 变化都触发 re-render。v0.5.1 接受，store 变化
  频率低，session 数小，不影响体验
- **未做 input 防抖 / 重复提交保护**：mkdir/rename 的"确认"按钮在 await 期间
  仍可点击；后端幂等性靠 sftp 协议层（同名 mkdir 会报错，rename 同名会报 source
  not exist），错误走 toast 路径。v0.6+ 加 `busy` 状态禁用按钮

## 教训

- **SftpBrowser 内部 useState 数量**：8 个 useState + 1 个 useRef（stale token），
  对一个 Modal 来说偏多但每个职责清晰（path / entries / loading / error /
  selectedPath / sortKey / sortDir / dialog / viewer）。如果 v0.6+ 加上传 / 搜索
  / 过滤，建议把子对话框状态和 viewer 状态独立成更小的 hooks，避免单组件膨胀
- **stale response token** 是处理"快速切换目录"的金标准。SftpList 是 async，没
  token 的话旧响应覆盖新结果（经典的 async race）。v0.6+ 上传 / 下载进度回调
  也会用相同模式
- **Wails binding 扩展 vs shim**：方案 B（扩展 App.d.ts）更接近 wails 真实生成结果，
  主理跑 `wails generate` 时 diff 干净。shim 方案（独立 `frontend/src/wails/sftp.d.ts`）
  会让 v0.5.1 整合时多一个 "把这行 import 改回去" 的小动作
- **icons 全用 lucide-react**：`lucide-react` 已经在 `package.json`（v0.5.0 加的），
  直接 import 即可，不需要 emoji / inline SVG 兜底
- **校验 / 失败处理在 onDialogSubmit 一处收口**：mkdir / rename / delete 三个分支
  共享同一个 `try/catch`，避免在 5 个不同的地方重复写错误 toast。v0.6+ 加 busy
  状态时也只改这一处
